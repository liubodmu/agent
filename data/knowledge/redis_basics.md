# Redis 面试知识点

## Redis 是什么

Redis（Remote Dictionary Server）是一个开源的、基于内存的键值对数据库。

特点：
- 高性能：基于内存，读写速度极快（10w+ QPS）
- 丰富的数据结构：String、List、Hash、Set、ZSet
- 持久化：RDB + AOF
- 单线程模型（Redis 6.0 后 IO 多线程，执行仍单线程）

## Redis 五种基本数据类型

### String（字符串）
- 最基本的类型，最大 512MB
- 应用：缓存、计数器、分布式锁

### List（列表）
- 双向链表实现
- 应用：消息队列、最新列表

### Hash（哈希）
- 类似 HashMap
- 应用：存储对象

### Set（集合）
- 无序不重复
- 应用：去重、共同好友（交集）

### ZSet（有序集合）
- 带分数的 Set，按分数排序
- 底层：跳表 + 哈希表
- 应用：排行榜、延时队列

## Redis 持久化

### RDB（快照）
- 定时将内存数据 dump 到磁盘
- 优点：恢复速度快，文件紧凑
- 缺点：可能丢失最后一次快照后的数据

### AOF（追加日志）
- 记录每次写操作命令
- 优点：数据安全性高
- 缺点：文件大，恢复慢

### 混合持久化（Redis 4.0+）
- RDB + AOF 结合，兼顾性能和安全

## Redis 缓存问题

### 缓存穿透
- 问题：查询不存在的数据，缓存和数据库都没有
- 方案：布隆过滤器、缓存空值

### 缓存击穿
- 问题：热点 key 过期，大量请求直接打到数据库
- 方案：互斥锁、永不过期 + 异步更新

### 缓存雪崩
- 问题：大量 key 同时过期
- 方案：过期时间加随机值、集群部署、限流降级

## Redis 分布式锁

基本实现：
```
SET key value NX EX 30
```
- NX：key 不存在才设置（保证互斥）
- EX：设置过期时间（防止死锁）

问题与优化：
- 锁过期但业务未完成 → Redisson 的看门狗机制
- 主从切换导致锁丢失 → RedLock 算法

## 面试常见追问

Q: Redis 为什么快？
A: 1.基于内存 2.单线程无锁竞争 3.IO多路复用 4.高效数据结构

Q: Redis 单线程为什么能处理高并发？
A: 单线程指的是命令执行单线程，避免了锁开销。网络IO用多路复用（epoll），一个线程处理多个连接。

Q: Redis 和 Memcached 的区别？
A: Redis 数据类型更丰富、支持持久化、支持集群、支持发布订阅。
